Arquitectura del módulo y dependencias

Arquitectura y dependencias

El módulo funciona desde el día uno con solo Drupal core. Las features adicionales se activan instalando los módulos contrib correspondientes, sin romper el funcionamiento base.

🏗️ Capas de dependencias

Capa 1 — Módulos de Drupal core (siempre disponibles)

node · taxonomy · media · field · text · options · image · file · filter · content_moderation

Forman parte de Drupal 11 core. No requieren instalación separada.

Capa 2 — Dependencias requeridas (el módulo no funciona sin ellas)

Ninguna dependencia contrib obligatoria. El módulo base funciona únicamente con Drupal core.

Capa 3 — Dependencias sugeridas (activan features opcionales)

Se declaran en composer.json como "suggest". El módulo las detecta en tiempo de ejecución y activa la feature correspondiente si están presentes.

📦 Features opcionales y sus módulos

Cada feature se activa instalando sus módulos con Composer. Si los módulos no están, la feature simplemente no aparece — el resto del módulo funciona con normalidad.

🗺️
Mapa de propiedades

Campo de geolocalización en el inmueble. Mapa interactivo con OpenStreetMap. Geocodificación automática de la dirección.

composer require drupal/geofield drupal/leaflet drupal/geocoder
🔎
Búsqueda con filtros facetados

Filtrado por tipo, precio, ciudad, habitaciones, etc. con conteo de resultados por opción.

composer require drupal/search_api drupal/facets
📊
Schema.org / JSON-LD (SEO)

Datos estructurados tipo RealEstateListing para Google. Mejora el posicionamiento en resultados de búsqueda.

composer require drupal/schemadotorg
📄
Ficha PDF del inmueble

Genera un PDF descargable con los datos del inmueble. Sin dependencias de servidor externo.

composer require drupal/entity_print tecnickcom/tcpdf
📬
Formulario de consulta / contacto de agente

Formulario "Me interesa este inmueble" y formulario de contacto del agente.

composer require drupal/webform
🗺️
Sitemap XML + imágenes

Genera el sitemap para buscadores incluyendo las imágenes de los inmuebles.

composer require drupal/simple_sitemap
🔄
Workflow de estado del inmueble

Estados: Disponible → En negociación → Arrendado / Vendido → Archivado. Módulo de Drupal core, solo hay que activarlo.

drush en content_moderation workflows

🖥️ Página de estado del módulo

El módulo incluye una página de administración en Administración → Configuración → Property Manager → Estado que muestra el estado de cada feature: activa, inactiva o incompleta. Para las inactivas muestra el comando exacto para instalarla.

💡 Al hacer clic en una feature "No instalado", la página muestra el comando composer require exacto y un enlace a la documentación del módulo. Al hacer clic en "Parcial", muestra qué módulo falta y cómo instalarlo.

⚙️ Detección de módulos en tiempo de ejecución

El módulo property_manager detecta qué módulos opcionales están activos usando el servicio module_handler de Drupal:

\Drupal::moduleHandler()->moduleExists('geofield')
  → true  → agrega campo de coordenadas al formulario
  → false → el campo no aparece, sin errores

Esto aplica para cada feature opcional. La lógica vive en hooks de property_manager.module y en los plugins de campo y formatter correspondientes.
