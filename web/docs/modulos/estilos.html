Estilos CSS del módulo

Hojas de estilo

El módulo tiene cuatro archivos CSS, cada uno asociado a una librería Drupal distinta. Dependiendo de dónde quieras hacer cambios visuales, sabrás exactamente qué archivo editar.

🗺️ Mapa general de estilos

Archivo Librería Dónde se carga Propósito
css/property_manager.form.css property_manager/form Vista edición Layout de campos en el formulario del Inmueble: etiquetas inline, íconos input-group, checkboxes horizontales, sección bloque.
css/property_manager.htabs.css property_manager/form + settings Vista edición Admin Transforma las pestañas verticales de Claro en pestañas horizontales estilo pill. Aplica en el formulario del Inmueble y en la página de Ajustes.
css/property_manager.node.css property_manager/node Vista pública Estilos de la vista pública del Inmueble: amenidades como tags con ✓/✗, precio destacado. Solo se carga al visualizar un nodo inmueble.
css/property_manager.admin.css property_manager/settings + status Admin Estilos de la página de Ajustes: grid de 3 columnas para el código automático, preview del formato, separadores.
💡 La librería form se adjunta en property_manager_form_alter() en property_manager.module. La librería node se adjunta en property_manager_node_view(). Las librerías están declaradas en property_manager.libraries.yml.

✏️ Vista de edición — property_manager.form.css

Todos los estilos del formulario de crear/editar Inmueble. Los selectores usan la clase .pm-tab-pane como scope para no afectar otros formularios de Drupal.

Sección en el archivoQué haceClave CSS
Bloque de código El aviso azul/gris con el código del inmueble en la parte superior del formulario. .pm-codigo, .pm-codigo--pending, .pm-codigo--assigned
Etiquetas inline Pone la etiqueta de cada campo a la izquierda (ancho fijo 10rem) y el input a la derecha en la misma fila. Aplica a select, text, url, number y entity-autocomplete. .pm-tab-pane :is(.form-type--select, ...)
Íconos input-group El addon izquierdo con ícono Font Awesome pegado al input, simulando el input-group de Bootstrap. Usa las variables de borde de Claro para que el ícono sea idéntico al borde del input. .pm-tab-pane .field-prefix:has(.pm-ficon)
Tab bloque En la pestaña Multimedia la etiqueta va en su propia fila (arriba del input) en lugar de a la izquierda. Aplica cuando el details tiene la clase pm-tab-block. .pm-tab-block :is(...) > .form-item__label
Checkboxes horizontales Las casillas de Características interiores y exteriores fluyen de izquierda a derecha. Resetea el posicionamiento float de Claro que causa problemas en flex. .pm-tab-pane .form-checkboxes, .form-checkboxes > .form-type--checkbox .form-boolean

Cómo cambiar el ancho de las etiquetas

Las etiquetas usan flex: 0 0 10rem. Para hacerlas más anchas o estrechas, cambia el valor en dos lugares:

/* property_manager.form.css — busca estas dos reglas */ .pm-tab-pane :is(...) > .form-item__label { flex: 0 0 10rem; /* ← cambiar aquí */ } .pm-tab-pane :is(...) > .form-item__description { padding-left: calc(10rem + 0.5rem); /* ← y aquí */ }

Cómo cambiar el spacing entre checkboxes

.pm-tab-pane .form-checkboxes { gap: 0.35rem 1.75rem; /* vertical horizontal */ }

Para cambiar un ícono

Edita el array en property_manager.module y cambia el nombre de la clase FA. Puedes buscar el nombre en fontawesome.com/icons (usa solo íconos de Free / Solid).

// property_manager.module — función _property_manager_add_field_icons() $icons = [ 'field_precio' => 'fa-dollar-sign', // ← cambia 'fa-dollar-sign' por cualquier ícono FA Free Solid 'field_habitaciones' => 'fa-bed', // ... ];

Para añadir un ícono a un campo nuevo

Agrega una línea al array. La función detecta automáticamente el tipo de widget:

$icons = [ // campos existentes... 'field_mi_nuevo_campo' => 'fa-home', // ← añadir aquí ];
La función auto-detecta si el campo usa widget directo (select), valor simple (number/text) o autocomplete de entidad (entity_reference_autocomplete). No necesitas especificarlo.

Versión de Font Awesome

Se carga desde CDN. Para cambiar la versión, edita property_manager.libraries.yml:

# property_manager.libraries.yml form: css: theme: https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.2/css/all.min.css: { type: external, minified: true } # ↑ Cambia 6.5.2 por la versión deseada. Consulta: cdnjs.com/libraries/font-awesome

🗂️ Pestañas horizontales — property_manager.htabs.css

Transforma el layout vertical de Claro en pestañas pill horizontales. Aplica en dos formularios (scoped por ID):

  • #property-manager-settings-form — Página de Ajustes del módulo
  • #node-inmueble-form y #node-inmueble-edit-form — Formulario de Inmueble
Qué cambiarSelector CSSPropiedad
Color de la pestaña activa .is-selected .vertical-tabs__menu-link background-color: var(--color-absolutezero) → cambia el color
Forma de los extremos (pill) .vertical-tabs__menu-link border-radius: 100px → reduce para esquinas más rectas
Tamaño de texto en formulario Inmueble #node-inmueble-form .vertical-tabs__menu-link font-size: 0.82rem
Pestaña activa por defecto Se configura en Administración → Property Manager → Ajustes (no en CSS). Ver sección Secciones del formulario.

🌐 Vista pública — property_manager.node.css

Solo se carga cuando se visualiza un nodo de tipo Inmueble (vía hook_node_view). No afecta al formulario de edición.

ElementoSelectorDescripción
Amenidades activas (✓) .pm-caract--yes Tag verde con check. Fondo #f0faf2, borde #c3e6cb, texto #155724.
Amenidades inactivas (✗) .pm-caract--no Tag gris neutro. Fondo #f8f9fa, borde #dee2e6, texto #6c757d.
Precio .field--name-field-precio .field__item Tamaño 1.4em, negrita, color azul oscuro #1a237e.
Lista de amenidades .field--name-field-caract-int .field__items Flex wrap con gap: 6px. Los tags se adaptan al ancho disponible.

Ejemplo: cambiar colores de las amenidades

/* property_manager.node.css */ /* Disponible — actualmente verde */ .field--name-field-caract-int .pm-caract--yes { background: #e8f4fd; /* azul claro */ border: 1px solid #b8daff; color: #004085; } /* No disponible — puedes ocultarlos completamente */ .field--name-field-caract-int .pm-caract--no { display: none; /* ← ocultar los que no aplican */ }

Ejemplo: cambiar estilo del precio

/* property_manager.node.css */ .field--name-field-precio .field__item { font-size: 1.6em; /* más grande */ color: #2e7d32; /* verde en lugar de azul */ font-weight: 700; }
💡 La clase .pm-caract--yes / .pm-caract--no se asigna dinámicamente en property_manager_preprocess_field() en property_manager.module. El módulo muestra todos los valores posibles (no solo los seleccionados), marcando cada uno como presente o ausente.

⚙️ Secciones del formulario — Configuración desde la UI

Las pestañas del formulario de Inmueble (Básica, Ubicación, Características…) son totalmente configurables desde la interfaz, sin tocar código.

AjusteDóndeNotas
Renombrar una pestaña Administración → Property Manager → Ajustes
/admin/config/property-manager/settings		 En la tabla de secciones, edita directamente el campo de etiqueta.
Cambiar la pestaña activa al abrir el formulario Selector «Pestaña activa por defecto» al final de la sección Secciones del formulario.
Mover un campo a otra pestaña En la tabla de la sección Ubicación de campos, cambia la columna «Sección» del campo deseado.
Reordenar los campos dentro de una pestaña Usa la columna «Peso» en la tabla de campos (menor peso = aparece antes).

Pestaña Multimedia — tab de bloque

La pestaña Multimedia tiene la clase CSS pm-tab-block además de pm-tab-pane. Esto hace que las etiquetas se muestren encima del input (en lugar de inline) para que sean consistentes con el widget de media que no tiene etiqueta lateral. Si mueves el campo field_video a otra pestaña, el comportamiento inline se restaura automáticamente en esa pestaña.

Para aplicar este comportamiento a otra pestaña, edita el array $block_tabs en _property_manager_build_form_sections() dentro de property_manager.module:

// property_manager.module — _property_manager_build_form_sections() $block_tabs = ['multimedia', 'anexos']; // añade el id de la sección aquí
