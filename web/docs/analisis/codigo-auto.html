Campo Código — Generación automática

Campo Código — Generación automática

El campo Código identifica de forma única cada inmueble. Se genera automáticamente al crear el nodo, sin intervención del usuario.

⚙️ Comportamiento esperado

ParámetroDescripción
Prefijo Texto libre definido por el administrador (ej: PROP, INM, VNT). Sin incluir el separador.
Separador Carácter opcional entre el prefijo y los dígitos. El administrador lo define libremente o lo deja vacío. Ejemplos: - → PROP-00001 / vacío → PROP00001 / _ → PROP_00001.
Dígitos Número de dígitos del contador, definido por el administrador (ej: 5 → 00001). Se rellena con ceros a la izquierda.
Secuencia Auto-incremental. Cada nuevo inmueble toma el siguiente número disponible.
Resultado Código = Prefijo + Separador + Número. Ejemplos: PROP-00001, PROP00001, INM_00042.
Editable El campo no se muestra en el formulario de creación/edición. Se genera en el hook presave.
Único No se reutiliza aunque el inmueble sea eliminado.

🖥️ Página de configuración del módulo

El administrador configura el comportamiento del código desde Administración → Configuración → Drupal Property Manager → Ajustes de código.

AjusteTipoValor por defecto
Prefijo del códigoTextoPROP
SeparadorTexto (1 carácter, puede quedar vacío)-
Número de dígitosEntero (1–10)5
Último número usadoEntero (interno, State API)0
Vista previa en tiempo real: al cambiar cualquier ajuste, la página muestra un ejemplo del código resultante antes de guardar.

🔧 Implementación técnica

  • Config form en src/Form/InmoCodigoSettingsForm.php
  • Valores guardados en config('property_manager.settings')
  • Generación del código en hook_node_presave() dentro de property_manager.module
  • El contador se almacena en el state de Drupal (state('property_manager.ultimo_codigo')), no en config (así no se exporta accidentalmente)
  • El campo field_codigo se marca como no editable en el formulario vía hook_form_alter()
✅ Decisión de diseño: El contador se guarda en el State de Drupal (no en Config) para evitar que se exporte/importe por error entre entornos y se resetee la secuencia en producción.
