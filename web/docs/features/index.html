Investigación de features y módulos

Features adicionales

Investigación realizada el 07/06/2026 sobre qué utilizan otros proyectos inmobiliarios en Drupal 11 y qué se puede incorporar para agregar valor. Organizado por prioridad para su revisión.

📌 Los módulos Metatag y Pathauto ya están instalados en este proyecto. El resto están disponibles en Drupal.org con compatibilidad D11 confirmada.

P1 — Diferenciadores inmediatos

Lo que ya espera el usuario al visitar una plataforma inmobiliaria moderna.

P1 · Geolocalización

🗺️ Mapa de propiedades

Cada inmueble se ubica en un mapa interactivo. El agente ingresa la dirección y el sistema la geocodifica automáticamente sin intervención adicional.

Módulos: geofield + leaflet + geocoder

Mapa: OpenStreetMap — gratis, sin API key.

Esfuerzo medio

Ver análisis detallado →

P1 · Búsqueda

🔎 Búsqueda con filtros facetados

El usuario filtra por tipo, precio, habitaciones, ciudad, etc. Los filtros muestran el conteo de resultados ("23 casas en Neiva"). Esencial para la UX de un portal inmobiliario.

Módulos: search_api + facets + backend de base de datos

Esfuerzo medio

Ver análisis detallado →

P1 · Galería

🖼️ Galería con lightbox

Las fotos del inmueble se muestran en una galería que se abre en modo lightbox al hacer clic. Expectativa estándar del usuario.

Módulos: Media (core D11) + photoswipe

Esfuerzo bajo
P1 · SEO

📊 Schema.org para inmuebles

Genera datos estructurados en JSON-LD con el tipo RealEstateListing. Google los usa para enriquecer resultados de búsqueda. Mapea directamente los campos que ya tenemos: precio, dirección, habitaciones, área, imágenes.

Módulo: schemadotorg (Schema.org Blueprints)

P2 — Alto valor, siguiente capa

Agregan diferenciación y mejoran la retención y conversión de usuarios.

P2 · Engagement

❤️ Favoritos

El usuario guarda los inmuebles que le interesan. Página "Mis favoritos" con la lista guardada. Mejora la retención y la intención de compra.

Módulo: flag

Esfuerzo bajo
P2 · Distribución

📢 Compartir en redes sociales

Botones para compartir el inmueble en WhatsApp, Facebook, X, Email, etc. Más de 100 redes disponibles.

Módulo: addtoany

Esfuerzo bajo
P2 · Documentos

📄 Ficha PDF del inmueble

El agente o el visitante descarga una ficha PDF con los datos del inmueble. Genera el PDF desde una vista "impresión" del nodo.

Módulos: entity_print + Dompdf (librería PHP, sin dependencias externas)

Esfuerzo medio
P2 · Presentación

💰 Formato de precio

El precio se muestra con separadores de miles y símbolo de moneda según el locale del sitio. Ej: $ 450.000.000 o € 250.000.

Implementación: Formatter personalizado (PHP NumberFormatter con intl)

Esfuerzo bajo
P2 · Leads

📬 Formulario de consulta

Formulario "Me interesa este inmueble" en cada ficha. Envía email al agente con los datos del interesado y el inmueble. Ya planeado para el tipo de contenido Agente.

Módulo: webform (planeado)

Esfuerzo bajo
P2 · SEO

🗺️ Sitemap XML + imágenes

Genera el sitemap XML incluyendo imágenes de los inmuebles. Esencial para indexación en Google.

Módulo: simple_sitemap

Esfuerzo bajo

P3 — Para fases posteriores

Funcionalidades avanzadas que agregan valor pero requieren más planificación o tienen mayor complejidad.

P3 · Engagement

🔔 Alertas de nuevos inmuebles

El usuario guarda su búsqueda y recibe un email cuando se publica un nuevo inmueble que coincide. Componente más complejo: requiere almacenar criterios por usuario y un cron que compare.

Módulos: eca + desarrollo personalizado

Esfuerzo alto
P3 · Herramienta

🏦 Calculadora de hipoteca / arriendo

Widget interactivo en la ficha del inmueble. El usuario ingresa cuota inicial y plazo y ve la cuota mensual estimada.

Implementación: Bloque JS embebido. Sin módulo adicional.

Esfuerzo bajo
P3 · Comparación

⚖️ Comparar propiedades

El usuario selecciona 2–3 inmuebles y los ve comparados lado a lado en una tabla. La selección se gestiona con Flag, la vista comparativa requiere desarrollo.

Módulos: flag + página personalizada

Esfuerzo medio-alto
P3 · Multimedia

🎥 Tour virtual / 360°

Embed de Matterport, Kuula u otro servicio de tours 360°. Se almacena la URL como campo de texto y se renderiza en un iframe.

Implementación: Campo URL + Twig. Sin módulo extra.

Esfuerzo bajo
P3 · Operación

🔄 Estado / Workflow del inmueble

Transiciones de estado: Disponible → En negociación → Arrendado / Vendido → Archivado. Evita mostrar inmuebles ya cerrados.

Módulo: Content Moderation (core D11)

Esfuerzo bajo
P3 · Ventas

🔗 Integración CRM

Los leads del formulario de contacto se envían al CRM del cliente (HubSpot, Salesforce, Zoho). La opción más flexible es Webform + webhook a Zapier/Make.

Módulos: webform + webhook / hubspot / salesforce

Esfuerzo variable
